# OSM Data Extraction & Setup For Camera Location Resolutions

This is a guide to manually get extract OSM data and import it into the application database.
Final result should be an increase of ~83MB.

*To be automated into a pipeline and containerised*

# 1. Get Required Resources

## 1.1 Install Postgres with PostGIS

* Ensure PostGIS has been installed via **Stack Builder**. Install if required.
    * Categories > Spatial Extensions > *"PostGIS `3.x` Bundle for PostgreSQL `18`"*

## 1.2 Download binaries for following tools (or install)

* Osmfilter - https://wiki.openstreetmap.org/wiki/Osmfilter
* OsmConvert - https://wiki.openstreetmap.org/wiki/Osmconvert
* osm2pgsql - https://osm2pgsql.org/doc/install.html

## 1.3 Download OSM data & Style file

* australia-latest.osm.pbf - https://download.geofabrik.de/australia-oceania/australia.html
  _(Australia Extract From Planet OSM)_
* Default style file (from [LearnOSM](https://learnosm.org/en/osm-data/osm2pgsql/#get-the-style-file)
  guide) - [Download link](https://learnosm.org/files/default.style)

# 2. Convert & Filter OSM Data

## 2.1 Convert `.pbf` to `.o5m`

Converts and filters data within SA for a **>10x size reduction**

```shell
osmconvert australia.osm.pbf -o=sa.05m -v -B=<BOUNDARY_POLY_FILE> --complete-ways
```

* `-o` - output file
* `--complete-ways` - preserve entire road geometry
* `-v` - verbose

## 2.2 Filter OSM data

Filter the converted `sa.05m` to include only values with:

* Valid `highway` for `planet_osm_line` (roads)
* `boundary=administrative or place=suburb` for `planet_osm_ploygon` (suburbs)

```shell
osmfilter sa.05m --keep="highway= or boundary=administrative or place=suburb" -o=sa_filtered.osm
```

# 3. Create DB & Import OSM Data

## 3.1 Create DB With Required Extensions

```shell
psql -h localhost -U postgres -f "<CREATE_DB_SQL_SCRIPT_PATH>"
```

* `CREATE_DB_SQL_SCRIPT_PATH` - as the path to `create_osm_db.sql`

## 3.2 Import OSM Data

```shell
osm2pgsql -c --slim --hstore -d <DB_NAME> -U <USER> -W -H <HOST> -S <STYLE_FILE_PATH> <OSM_FILE_PATH>
```

* `osm2pgsql` - this can just be the `.exe` path
* `--hstore` may not need - keeps tags
* `-W` - Force password
* `OSM_FILE_PATH` - the file generated by `osmfilter` sa_filtered.osm

# 4. Transform OSM Data

The `canonise_supabase` script will:

* Create normalised and canonised tables:

    * `streets_canon` <-- `planet_osm_line`
    * `suburbs` <-- `planet_osm_polygon`

* Create table for camera location queries (street + suburb) `streets_by_suburb`

```shell
psql -h localhost -d <DATABASE_NAME> -U postgres -v ON_ERROR_STOP=1 -f "<CANONISE_OSM_SCRIPT_PATH>"
```

* `CANONISE_OSM_SCRIPT_PATH` - as the path to `canonise_osm_v1.sql`
* `-d` - should be `osm` (as created by `create_osm_db.sql`)

# 5. Create Dump file & Restore Into Target DB

## 5.1 Dump

```shell
pg_dump -Fc -t streets_canon -t suburbs osm2 > canonised_streets_suburbs -U postgres -h <HOST>
```

## 5.2 Restore

First ensure the required extensions are installed in the `target_database` by running the follwiing `psql`

```sql
CREATE EXTENSION IF NOT EXISTS hstore;
CREATE EXTENSION IF NOT EXISTS postgis;
```

Restore the dump file to the `target_database`.

```shell
pg_restore --no-owner -U postgres -h <HOST> -d target_database <DUMP_FILE>
```
